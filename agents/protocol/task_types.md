# 작업 유형별 담당자 라우팅 테이블

---

## 라우팅 원칙

```
판단 필요? → Claude
검색 필요? → GPT
승인 필요? → 사용자
```

## 경계선이 애매한 경우

### "타입 정의를 GPT가 생성 → Claude가 검토"
- GPT: 테이블 구조 보고 TypeScript 타입 초안 작성
- Claude: 비즈니스 로직 맞게 수정 후 최종 적용

### "외부 API 연동 코드"
- GPT: API 스펙 문서 확인, 예제 코드 제시
- Claude: 실제 구현 (보안 처리, 에러 핸들링, RLS 고려)

### "UI 컴포넌트"
- 비즈니스 로직 없는 순수 UI: GPT 가능
- 마진 계산기, 견적 마법사 등 로직 있는 것: Claude 직접

## 잘못된 라우팅 예시

❌ GPT에게: "마진 계산 로직 짜줘" → Claude 담당
❌ Claude에게: "Toss Payments 공식 문서에서 webhook 이벤트 목록 찾아줘" → GPT 담당
❌ 사용자가 직접: "코드 여기 이렇게 바꿔야겠다" → Claude에게 의뢰
