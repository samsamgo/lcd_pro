# 오케스트레이터(사용자) 운영 가이드

> 이 문서는 Claude + GPT 멀티에이전트 팀을 운영하는 사용자를 위한 가이드다.

---

## 역할 요약

| 역할 | 담당 | 모델 |
|------|------|------|
| 오케스트레이터 | 방향 설정, 최종 승인, 실행 트리거 | 사용자 |
| 리드 아키텍트 | 설계, 코드, 디버깅, 의사결정 | Claude |
| 리서처 | 검색, 외부 API, 반복 작업 | GPT Agent |

---

## 일상 운영 워크플로

### 새 대화 시작 시 (Claude에게)
```
agents/prompts/claude.md 읽고 역할 시작해
```
또는 claude.md 파일 내용을 붙여넣는다.

### 작업 요청 방법
- **Claude에게**: 비즈니스 결정, 코드 구현, 버그 수정, 아키텍처 질문
- **GPT에게**: 시장조사, 문서 검색, 외부 API 확인, 반복 데이터

### GPT 작업 결과를 Claude에게 전달할 때
GPT가 보낸 `[CLAUDE_RESULT]` 블록을 그대로 Claude 대화창에 붙여넣는다.

---

## 오케스트레이터가 직접 결정하는 것

- 기능 우선순위
- 출시 타이밍
- 비용 한도 (Supabase 요금, Vercel 플랜 등)
- 파트너/고객 관계
- 최종 UI/브랜드 방향

---

## 금지 사항

- Claude가 GPT_TASK를 발행했는데 무시하고 직접 GPT에게 다른 작업을 주는 것
  → 두 에이전트의 맥락이 분리된다
- 같은 작업을 두 에이전트에게 동시에 시키는 것
  → 중복 결과, 혼선 발생

---

## CURRENT.md 관리

`agents/context/CURRENT.md`는 프로젝트의 살아있는 상태 문서다.

- Claude가 세션 종료 시 업데이트한다
- 다음 세션 시작 시 Claude가 이 파일을 먼저 읽는다
- 사용자가 직접 "다음에 이어서 해" 같은 상태를 여기에 메모해도 된다

---

## 긴급 상황 처리

배포 실패, 데이터 손실 위험, 보안 이슈가 발생하면:
1. 현재 작업을 중단한다
2. Claude에게 "긴급: [증상]" 로 시작하는 메시지를 보낸다
3. Claude가 `agents/protocol/escalation.md` 기준으로 처리한다
